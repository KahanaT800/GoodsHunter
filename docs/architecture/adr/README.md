# Architecture Decision Records (ADR)

本目录包含 GoodsHunter 项目的架构决策记录。

---

## 什么是 ADR？

架构决策记录 (Architecture Decision Record) 是一种轻量级文档，记录项目中重要的技术决策，包括：

- **背景**: 为什么需要做这个决策
- **选项**: 考虑过哪些方案
- **决策**: 最终选择了什么
- **后果**: 这个决策带来的影响

## ADR 模板

创建新的 ADR 时，请使用以下模板：

```markdown
# ADR-XXXX: 标题

## 状态

提议 / 已采纳 / 已废弃 / 已取代

## 背景

描述问题背景和需要做决策的原因。

## 决策

描述做出的决策。

## 考虑的选项

### 选项 1: xxx

**优点:**
- ...

**缺点:**
- ...

### 选项 2: xxx

...

## 后果

### 正面

- ...

### 负面

- ...

## 参考

- 相关链接
```

---

## 状态说明

| 状态 | 说明 |
|------|------|
| 提议 | 正在讨论中，尚未最终确定 |
| 已采纳 | 已被接受并实施 |
| 已废弃 | 不再相关或有效 |
| 已取代 | 被新的 ADR 取代 |
